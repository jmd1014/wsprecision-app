# 조달 분류 (도급 / 사급) 정책

우성정밀 제품은 조달 방식에 따라 **도급** 또는 **사급** 으로 구분된다.
같은 기능의 제품이라도 조달이 다르면 분기되어 다른 PN 으로 관리하는 경우가 있다.

⚠️ **PN 접두사(4S, S 등)는 재질 구분 컨벤션이며 도급/사급 식별자가 아니다.** 자동 추론 금지.

---

## 1. 정의

| 구분 | 의미 | 원가 구성 | 매출가에 포함 |
|---|---|---|---|
| **도급** | 우성정밀이 소재 구매 + 가공 + 납품 | 소재비 + 외주 + 열처리 + 표면 + ... | 전체 |
| **사급** | 고객이 소재 공급 + 우성정밀이 가공만 | 0(소재) + 외주 + 열처리 + 표면 + ... | 가공비만 |

---

## 2. PN 접두사 — **재질 구분** (도급/사급 무관)

| 접두사 | 의미 | 적용 범위 |
|---|---|---|
| `4S`, `S`, ... | **재질 구분** (이전부터 사용해온 명명 컨벤션) | 모든 제품 |

→ 접두사는 **절대 임의 변경 금지**. 도급/사급 자동 추론에 사용 금지.

---

## 3. 도급 전환 특수 케이스

**AHYBV 시리즈 본넷 (20AHYBV, 40AHYBV, 80AHYBV, 150AHYBV) 에 한해서만** 도급 전환 시 PN 분기 관행이 있었음:

```
사급 (원본 PN):           20AHYBV-XX, 40AHYBV-XX, 80AHYBV-XX, 150AHYBV-XX
도급 전환 시 분기 PN:      4S20AHYBV-XX, 4S40AHYBV-XX, ...
  ↑ AHYBV 본넷에 한정. 다른 제품군의 4S 와는 무관.
```

→ **이건 일반 규칙이 아닌 특수 케이스**. 다른 제품군의 4S 접두사는 단순 재질 구분.

---

## 4. 조달 분류 메타데이터 (products 테이블)

| 컬럼 | 값 |
|---|---|
| `procurement_type` | `'도급'` 또는 `'사급'` — **사용자가 명시 지정** |
| `procurement_start_date` | 분류 변경 시작일 (시점 분기) |
| `procurement_prev_type` | 이전 분류 (변경 추적) |

→ 자동 추론/일괄 적용 X. 사용자가 제품별로 직접 지정.

---

## 5. BOM 처리

### 도급
- BOM 자재행: 정확히 입력 (qty_per_pc, shared_factor, material_id)
- 자재 단가: `products.material_unit_price` 또는 매입 매핑
- BOM 공정행: 외주/열처리 LOT 단가 입력
- 추정원가 = 소재 + 모든 공정

### 사급
- BOM 자재행: 자재 정보 입력 (참고용 — 어떤 자재를 쓰는지 추적)
- 자재 단가: **0 또는 NULL** (소재비 미산입)
- BOM 공정행: 외주/열처리 LOT 단가 입력
- 추정원가 = 공정만 (소재 제외)

---

## 6. 화면 동작

### 🎯 TOP 우선 정비
- `procurement_type` 미설정 → "⚠️ 분류 미설정 — 사용자가 직접 지정" 표시
- **자동 추론 금지** (4S 접두사 등은 재질 구분이라 무시)
- **사급** 인 경우:
  - 자재 단가 입력 영역 비활성화 (정보 안내)
  - 이미 단가가 입력되어 있으면 **"0으로 초기화"** 버튼 노출
- **도급** 인 경우:
  - 자재 단가 입력 영역 활성 (필수 체크리스트)
  - 현재 값 위에 새 값 덮어쓰기 가능 (정정)

### 마스터 관리 → 🧱 제품 편집
- `procurement_type` 컬럼 직접 편집 가능
- 변경 시 자동으로 변경 이력 기록 (procurement_prev_type, procurement_start_date)

---

## 7. PN 명명 — 손대지 말 것

다음은 모두 **재질 / 분류 컨벤션** 으로, **변경 금지**:
- `4S`, `S` 같은 접두사
- `-XX`, `-XXX` 같은 일련번호 / 사양 코드
- 제품군 코드 (`AHYBV`, `MRG6`, `FLANGE` 등)

→ PN 은 거래처 발주서 / 매입 / 매출 / BOM 모든 매핑의 기준이므로 절대 임의 변경 금지.

---

## 8. 변경/정정 절차

1. **조달 분류 정정**: 마스터 관리 → 제품 편집 / TOP 정비 페이지에서 procurement_type 수정
2. **사급인데 자재 단가 잘못 입력됨**: TOP 정비에서 "🗑 0 으로 초기화" 버튼
3. **PN 변경**: **금지** (거래 매핑 깨짐)
4. **새 PN 생성** (조달 전환 등 부득이한 경우): 마스터 관리 → 제품 편집 → 신규 추가

---

## 9. 자동 처리 금지 사항

| 시도 | 금지 사유 |
|---|---|
| 4S 접두사 → 도급 자동 지정 | 4S 는 재질 구분이지 도급 식별자가 아님 |
| 일반 PN → 사급 자동 지정 | 도급일 수도 있음. 사용자 확인 필수 |
| 일괄 procurement_type 부여 | 제품별로 사용자가 판단해야 함 |
| PN 패턴 매칭 → 분류 변경 | 도메인 규칙은 사람만 안다 |

→ 모든 분류는 사용자가 명시 지정. 앱은 **상태 표시 + 입력 도구**만 제공.

---

## 10. 검증 SQL — 현황 파악 (자동 적용 X)

```sql
-- procurement_type 설정 현황 (활성 제품)
SELECT
  COALESCE(procurement_type, '(미설정)') AS proc_type,
  COUNT(*) AS cnt
FROM products
WHERE archived_at IS NULL
GROUP BY 1
ORDER BY cnt DESC;
```

→ 미설정 건수가 많으면 사용자가 직접 한 건씩 지정 (혹은 정책상 미설정 유지).
